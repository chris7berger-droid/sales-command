WTC CALCULATOR ARCHITECTURE (key facts)
====================================================

- WTCCalculator.jsx lives in src/pages/ NOT src/components/
- Props: proposalId (UUID), wtcId: wtcIdProp (UUID or null), workTypeId, onClose
- wtcId is STATE: const [wtcId, setWtcId] = useState(wtcIdProp)
- autosaveTimer = useRef(null) — debounced 1.5s autosave on any state change
- First change on new WTC creates Supabase record automatically
- const [saved, setSaved] = useState(!!wtcIdProp) — still present for lock state
- work_type_id is INTEGER (1-40), not UUID
- WTC has 7 tabs: Bidding Info, Labor, Materials, Scope of Work, Travel, Discount, Summary
- Summary tab has 2-step action flow: Lock & Approve → Generate PDF
- PDFPreviewModal is built inside WTCCalculator.jsx (function PDFPreviewModal at line ~1246)
- SOW auto-loads from work_type_sow_templates only if sales_sow is currently empty
- On autosave: writes proposalPrice back to proposals.total
- Travel state fields: drive_rate, drive_miles, fly_rate, fly_tickets,
  stay_rate, stay_nights, per_diem_rate, per_diem_days, per_diem_crew
- calcTravel(t) uses new field names
- Lock behavior: locked WTCs show amber banner + click blocker on tabs 1-6
- Unlock button on Summary tab unlocks without requiring Change Order

====================================================
DESIGN PHILOSOPHY
====================================================

- Favor fast obvious clicks over fewest clicks
- UI flows step by step, one question at a time
- Never overwhelm user with a full form at once
- buildStepList() architecture for wizards
- Change Orders always require full WTC
- PDF stays white background — customers print proposals, ink costs matter
- No teal text on brown/linen backgrounds — use dark #1c1814 instead
- Teal is for buttons, borders, badges, active states only
