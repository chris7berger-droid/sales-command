   - Sends branded email to billing contact with "Pay Now" button

   - Billing contact auto-resolved from proposal -> call_log -> customer

4. Stripe Webhook — stripe-webhook Edge Function:
   - Receives checkout.session.completed from Stripe
   - Updates invoice: status=Paid, stripe_checkout_id, stripe_payment_id, paid_at
   - Sends payment receipt email to customer
   - Sends payment notification email to sales rep
   - Signature verification via STRIPE_WEBHOOK_SECRET

   branded "Payment Received" confirmation page after Stripe checkout

6. Invoice Edit Mode (New status only):
   - Editable: invoice #, due date, discount, description/PO#, billing % per line
   - Save recalculates line amounts from updated billing percentages
   - Cancel discards changes

7. Invoice Pull Back — resets Sent/Waiting for Payment/Past Due to New:
   - Clears sent_at, stripe_checkout_id, stripe_payment_id, paid_at
   - Confirmation prompt before executing
   - Paid invoices are locked (no edit, no pull back)

8. Invoice Delete — double confirmation prompt:
   - Deletes invoice_lines first (FK children), then invoice
   - Available on all statuses

9. Auto-refresh — invoice detail polls every 5s for payment status

====================================================
NAVIGATION ARCHITECTURE
====================================================

App.jsx manages page state via setActive("pageName"):
- CallLog -> Proposals: onNewProposal sets initialProposal.job, switches to proposals
- CallLog -> Proposal detail: onNavigateProposal sets initialProposal.openId,
  Proposals.jsx load().then finds matching proposal and sets sel
- CallLog -> Invoices: onNavigateInvoice switches to invoices page
- Home -> CallLog: setActive("calllog") with optional setBidDueFilter

====================================================
INVOICE FLOW ARCHITECTURE
====================================================

Invoice List -> + New Invoice -> select Sold proposal -> set billing % per WTC
-> Create Invoice (auto-generates ID) -> Invoice Detail

Invoice Detail (New):
  -> Edit Invoice (change #, due date, discount, description/PO#, billing %)
  -> Preview / Send -> InvoicePDFModal (preview or send tab)
  -> Send Invoice -> send-invoice Edge Function -> Stripe Checkout session
  -> Email to billing contact with Pay Now button

Customer Payment:
  -> Clicks Pay Now -> Stripe Checkout page -> pays
  -> Redirected to /invoice-paid success page
  -> Stripe fires checkout.session.completed webhook
  -> stripe-webhook Edge Function -> updates invoice to Paid
  -> Sends receipt email to customer + notification to sales rep

Manual Payment (check):
  -> Mark as Paid button on Sent/Waiting for Payment/Past Due invoices

Pull Back:
  -> Resets to New, clears Stripe fields
  -> Can edit and resend

====================================================
SIGNING FLOW ARCHITECTURE (SC-30)
====================================================

Proposal Detail -> Send Proposal -> ProposalPDFModal (Send tab)
-> send-proposal Edge Function -> Resend email -> customer gets link
-> /sign/:token -> PublicSigningPage.jsx -> customer types name
-> jsPDF generates branded PDF -> signed-proposals bucket
-> pdf_url saved to proposal_signatures (with signed_at)
-> proposals.status = Sold, proposals.approved_at = now
-> call_log.stage = Sold
-> proposal-signed Edge Function -> Resend email to rep
-> Proposal Detail shows Download Signed PDF button

signing_token is generated client-side via crypto.randomUUID() on
proposal creation. PublicSigningPage validates UUID format before query.

INTERNAL APPROVE FLOW:
Proposal Detail -> Internal Approve button -> modal (approved_by, reason)
-> proposals.status = Sold, internal_approval = true, approved_by, approval_reason
-> call_log.stage = Sold
-> PDF shows Internal Approval block instead of signature lines
-> Download Approved PDF button opens PDF preview

PULL BACK FLOW:
Proposal Detail -> Pull Back button -> confirmation
-> proposals.status = Draft, clears approval fields
-> proposal_signatures deleted
-> proposal_wtc unlocked
-> call_log.stage = Wants Bid

====================================================
EDGE FUNCTIONS
====================================================

send-proposal (--no-verify-jwt):
  - Sends customer email with signing link
  - Sends rep notification (non-blocking)
  - Validates customerEmail is not empty
  - Checks Resend API response

proposal-signed (--no-verify-jwt):
  - Called from PublicSigningPage after customer signs
  - Sends rep email with signer name, date, job details
  - Non-blocking from signing flow

send-invoice (--no-verify-jwt):
  - Creates Stripe Checkout session
  - Sends customer email with Pay Now link
  - Validates customerEmail and amount
  - Returns checkoutId and checkoutUrl

stripe-webhook (--no-verify-jwt):
  - Receives Stripe checkout.session.completed events
  - Verifies webhook signature
  - Updates invoice to Paid with Stripe IDs and timestamp
  - Sends receipt email to customer
  - Sends notification email to sales rep
