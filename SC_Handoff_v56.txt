1. FIX: WTC sticky header / double scrollbar / content bleed (RESOLVED)
   - Bug: WTC page had two scrollbars, header scrolled off screen cutting off
     buttons, and content bled out of containers
   - Root cause: WTC used margin: "-28px -32px 0" to escape parent padding
     and height: "calc(100vh - 50px)" which created a second scroll container
     inside the already-scrollable data-app-content div
   - Fix: Added useEffect that sets padding: 0 and overflow: hidden on the
     parent data-app-content div when WTC mounts, restores on unmount.
     Removed negative margins and changed height to 100% so WTC fills the
     parent naturally with a single scroll container.
   - Result: One scrollbar, fixed header/tabs, no content bleeding
   - This bug persisted through multiple failed attempts in v55 (zIndex,
     isolation, flex layout). The clean fix was removing the hack (negative
     margins) and instead telling the parent to yield its padding.
