## Top 3 Urgent

1. **B11 — `team_members_update` self-elevation (CRITICAL).** `sql/rls_identity_tables.sql:64` still USES only `tenant_id = public.get_user_tenant_id()` in USING/WITH CHECK. Any Sales Rep flips their own `role` to `Admin`/`Manager` with a single PostgREST call, unlocking every role-gated money policy downstream. Companion HIGHs on `team_members_insert/_delete:60,69` and `tenant_config_update:94` widen the blast radius. Fix: `AND public.is_admin_or_manager()` on USING and WITH CHECK; if reps must self-edit non-role columns, add a whitelisted `update_my_profile()` SECURITY DEFINER RPC. Four audit cycles unfixed.
2. **B6 — `sql/rollback_drop_public_read_policies.sql:4-8` rebuilds the 2026-04-26 anti-pattern (CRITICAL).** Live in-repo SQL that reinstates `tenant_config_public_read USING (true)` and `team_members_public_read USING (active = true)` — every tenant's `stripe_customer_id`, `stripe_subscription_id`, billing goals, license number PLUS every active team member's email/phone/auth_id/role, exposed to the publishable anon key. Header says "Use only if get_public_tenant_config() / get_rep_contact() RPCs fail in production" — a nudge to run it in exactly the wrong direction. Fix: `git rm sql/rollback_drop_public_read_policies.sql`. Legitimate rollback is to revert the drop migration, not restore the incident's proximate cause.
3. **Invoices.jsx money handlers + WTCCalculator.jsx — four client-only gates on money writes (HIGH × 4, NEW).** `handleCreate` (`Invoices.jsx:220`) mints invoices with arbitrary `amount` against any readable proposal; `handleSaveEdit` (`:1591`) rewrites `invoices.amount`, `invoice_lines.amount`, retention, discount; `handleVoidConfirm` (`:1820`) triggers `qb-void-invoice` + wipes Stripe fields + optionally inserts a replacement invoice; `handleSave` (`WTCCalculator.jsx:1956`) rewrites `proposal_wtc` (labor, markup, materials) and `proposals.total`. None of these paths have a client role check; RLS is the only line of defense, and Pass B confirms the RLS UPDATE policies on `proposals`/`invoices`/`invoice_lines`/`proposal_wtc` — insofar as they still live in this repo — do not gate on Admin/Manager either. A Sales Rep with a browser devtools console can rewrite any tenant invoice or WTC. Fix: add `if (!canManage) return;` on the four handlers AND enforce `is_admin_or_manager() OR sales_name = current_user_name` in RLS on the affected UPDATE policies (in `command-suite-db`).
