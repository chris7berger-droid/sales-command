## Tier definitions

The `Tier` column replaces the prior H/M/L Pri field. Tier captures
severity √ó likelihood-soon √ó cost √ó strategic leverage as one decision:

- **T0** ‚Äî Drop everything. Active prod breakage, in-flight security incident.
- **T1** ‚Äî This session. High-severity-and-likely √ó low-cost. Quick wins on customer-facing surfaces.
- **T2** ‚Äî This sprint. High strategic leverage, or unblocks T1/T2 work.
- **T3** ‚Äî When convenient. Low-severity bugs, refactor, polish.
- **T4** ‚Äî Only if forced. Items needing a re-trigger before they can move (missing artifacts, blocked on external).

Dependency chains are surfaced with `Blocks: <ID>` / `Blocked by: <ID>`
as the first line of the Notes column.
