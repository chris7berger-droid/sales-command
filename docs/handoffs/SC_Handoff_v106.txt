DECISIONS / CHOICES MADE
────────────────────────
- Two-migration compat-safe rollout shape (A now, B after ≥48h).
  Audit terminal explicitly required this so in-flight customer
  browsers with cached old JS don't break during the deploy window.
  Migration A keeps the 1-arg `mark_proposal_signed` form AND the
  legacy `proposal_signatures_public_insert_token` policy alive; both
  are dropped in Migration B after deploy traffic verification.
- Sold revisit window = approved_at + 1 year. Compromise between
  "permanent audit trail" (5+ years) and "minimize long-tail public
  exposure" (90 days). 1 year covers bookkeeper revisit needs for the
  common case; older proposals will need a fresh link issued if
  re-access is needed.
- Drafts backfilled with `expires_at = now() + 90d` (not
  created_at + 90d). No customer has the link yet; `handleSend`
  refreshes expires_at before email send anyway. Cleaner than
  backfilling Drafts to past dates.
- ALREADY_SIGNED UX is silent. Stale-tab / double-click → page
  silently renders Accepted via `setSigned(true)`. Audit-approved.
- IP captured in edge function from `x-forwarded-for`, not React
  body. Verified end-to-end on prod (`76.235.216.194` for a real
  signing session). Audit fix #3.
- `mark_proposal_signed` 5-arg p_pdf_url uses regex interpolation
  of `v_proposal_id` (audit note L1, low-risk because IDs are
  UUIDs). Tracked as an optional follow-up; not a deploy blocker.
- `ProposalPDFModal.handleSend` writes `status='Sent'` BEFORE
  Resend fires (audit fix #1). Trade-off: if Resend fails after the
  UPDATE, the row is flagged Sent with refreshed expiry but no email
  delivered — the rep retries. Documented in deploy notes (audit
  note L2).
- 1-arg compat wrapper inlined as `RETURN QUERY SELECT * FROM
  mark_proposal_signed(p_token, NULL::text, NULL::text, NULL::text,
  NULL::text)`. 5-arg makes signature insert OPTIONAL (skipped when
  `p_signer_name` is NULL) so the wrapper path produces the same
  status/consumed/call_log flip without inserting a signature row
  (the old JS path inserts via the legacy anon policy).
- Two scratch projects spun up + deleted: `lvbdsfyppaogaezvqmrg`
  for the 16-scenario SQL smoke; `nqanzszlbbjkercgwrtl` for the
  R1–R9 rollback round-trip. Both required Docker for
  `supabase db dump --linked -s public` (DDL-only, no customer
  data crossed). Docker Desktop started mid-session per Chris's
  approval; left running.
