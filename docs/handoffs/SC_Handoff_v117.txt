   changes. The 1-arg wrapper is dropped first (it depends on
   the 5-arg), then the 5-arg, then both recreated with GRANT
   TO anon re-issued. Surfaced by A1-Code audit item #2.

   path uses fnData?.became_sold ?? true; RPC fallback path

   correctly during the deploy window when old RPC (without
   became_sold column) is still live. Edge fn path surfaced
   during build; RPC fallback path surfaced by A1-Code audit
   item #4.

3. O3 sequencing — O3 (drop 1-arg wrapper) must land AFTER
   this C1 migration. C1 updates both overloads; O3 drops the
   1-arg. The migration file timestamps enforce correct order
   (20260514 > 20260510). Confirmed safe in both orderings by
   audit item #7.

4. NOTIFY outside BEGIN/COMMIT — intentional. The NOTIFY fires
   after the DDL transaction commits, ensuring PostgREST reloads
   against the committed schema. Matches the pattern in
   20260510120000.
