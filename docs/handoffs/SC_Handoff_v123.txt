DECISIONS / CHOICES MADE
────────────────────────
1. Used USING (true) for work_types instead of a token-gated EXISTS
   pattern. work_types has no tenant_id, no sensitive data — it's a
   reference table of 40 work type names and cost codes. Token gating
   would be overhead with no security value.

2. Added an authenticated SELECT policy alongside the anon one. If
   RLS was previously disabled on work_types, enabling it without an
   authenticated policy would lock out the internal app. The DROP IF
   EXISTS + CREATE is idempotent either way.

3. Merged to main mid-session rather than holding the branch. Chris
   needed the email → invoice link flow to work on prod to verify,
   since the invoice email hardcodes salescommand.app as the domain.
   Preview deploys can't intercept that path.
