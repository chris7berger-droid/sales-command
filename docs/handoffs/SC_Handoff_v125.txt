SESSION SUMMARY
Out-of-cycle hotfix. User opened the session with a 500 error on the
"Send Pay App" flow (AGRU 230 Lyon, Pay App #1) — UI showed "Edge
Function returned a non-2xx status code." Audit traced the bug to the
C9 audit commit (e662d24, 2026-05-09): authenticateCaller() refactor
removed the `const authHeader = req.headers.get(...)` declaration at
the top of send-pay-app/serve(), but the fire-and-forget qb-sync-invoke
at line 466 still referenced the deleted variable. ReferenceError
thrown synchronously while building fetch options, caught by outer
try/catch, returned 500 — even though Resend send had already succeeded
and DB writes had already committed.

10-day silent regression. Email + DB state went through correctly on
every send in the window; only the QB sync was silently skipped.

Fix shipped, deployed, backfill complete.

AUDIT FINDINGS (preserved for future)
─────────────────────────────────────
Sweep across all 25+ edge functions found send-pay-app was the ONLY
file with a `authHeader` reference that wasn't declared in scope.
Every other usage (invite-user, deactivate-user, delete-user,
send-proposal, extract-sov, _shared/tenantAuth.ts) declares it locally.
stripe-webhook is the only other fn→fn caller — it uses service-role
for webhook context, different pattern, no conflict.

`git log --since=2026-05-08` on send-pay-app, qb-sync-invoice, and
_shared/tenantAuth.ts showed zero commits between C9 and the bug
report — confirmed nothing has built on top of the regression.

Lesson for future fn→fn auth refactors: grep for ALL references to
the variable being removed, not just the declaration site.
