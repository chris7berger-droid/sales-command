B38 SCOPE CUT — why the triggers are gone
- Trigger #1 (amount-changed staleness on Edit save) can't fire in
  current code. `Edit Invoice` is gated on `status === "New"`
  (Invoices.jsx:1603) and `handlePullBack` already nulls
  `stripe_payment_link_id` + calls `deactivate-payment-link`. So the
  only way to edit a Sent invoice is Pull Back → Edit → Send, which
  guarantees a fresh link minted on every send. No scenario exists
  where the customer holds a stale-amount link.
- Trigger #2 (Past Due pulse) was built in commit 36baeab then
  reverted in commit 976a772 after Chris caught that the underlying
  premise was wrong. Since send-invoice always mints a fresh link
  on send, there's no "stale link" scenario for a past-due invoice
  either — the customer's old email link is either (a) deactivated
  via pullback or (b) still valid because no one has resent yet. A
  pulse to nudge a resend doesn't surface any new info — it's just
  a "this is past due" reminder, which the existing red "2d overdue"
  pill on InvoiceDetail already provides.
- Trigger #3 (Stripe webhook on payment_link.deactivated) was already
  deferred to "phase 2" in the original B38 plan.
