=== Decisions / Choices Made ===
- A4 → Option A (remove the public-page Pay button; payer pays via email button).
  Option B (per-recipient viewing tokens) declined as a separate larger task — would
  fork the feature. Page is now view/print only.
- #1 → Option B + mandatory live display. For linked recipients, resolve the email
  LIVE from customer_contacts at send AND display, instead of the stored snapshot.
  A linked email is a current contact, so it passes the C9 allowlist by construction
  — fixes the stale-snapshot send-block WITHOUT weakening C9. Rejected Option A
  (propagate edits onto recipient rows): it would rewrite "who we sent to" on already-
  sent invoices and didn't cover deletes.
- #SEC1 → Level 2 (real boundary), then corrected to default-deny. We deliberately
  did NOT settle for "hide the column in the client select" (Level 1) — anon holds the
  key, RLS is row-level, so a hand-crafted request could still pull the column. The
  column-level REVOKE we first wrote was a no-op under the table grant; the working fix
  is REVOKE-all + GRANT only safe columns. LESSON BANKED as memory
  feedback_column_revoke_noop: never trust a column REVOKE under a table-level grant;
  use default-deny and smoke-verify as anon.
- #8 (shared RecipientsCard across invoice + proposal) DEFERRED — a ~419-line cross-
  surface refactor touching the shipped ProposalDetail inside a High-security batch.
  Shipped the security/correctness fixes clean; do the DRY extraction deliberately later.
