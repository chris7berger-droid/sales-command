----------------------------------------------------------------
SESSION SUMMARY
----------------------------------------------------------------
Fixed the long-standing bug where, after sending an invoice, the
"Edit" and "Sync to QuickBooks" buttons stayed visible as if the
invoice were never sent — clearing only after navigating away and
back. Root cause: local `inv` state was reconciled after send only
from a hardcoded optimistic object that omitted server-written fields
(esp. `qb_invoice_id`) and never refetched. Fixed in `Invoices.jsx`,
merged to main (PR #34), and prod-verified by a real standard send.

Separately, while testing, the QB "Connect to QuickBooks" flow on a
job named "3:35 bug hunt work" failed with an opaque "Edge Function
returned a non-2xx status code". Diagnosed to a colon in the job name
(QB forbids `:` in DisplayName). Chris took the rename workaround;
filed B57 with the full root cause + fix rather than fixing this
session.

CHANGES SHIPPED

PR #34 (squash) — main @ 24c7ba1
  "Fix invoice sent-state buttons not clearing until remount"
  - New `reloadInv()` helper in `InvoiceDetail` — refetch + merge the
