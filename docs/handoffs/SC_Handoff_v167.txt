SESSION SUMMARY
---------------
Pure UI session. Started from a parked /detach stream to recolor one stark-white checkbox (the
"Mark as the job's deposit invoice" box on the invoice detail). Root cause: a native
`<input type="checkbox">` always renders a browser-WHITE fill when unchecked, regardless of
`accentColor` — that only colors the checked state. Rather than patch inline, built one shared
white-free `Checkbox` component and swept ALL 15 native checkboxes in the app onto it, plus
folded the app's only other checkbox pattern (`ContactFlagToggle`) into it so there's a single
checkbox component. Then Chris flagged that four "rich cards" (pay-app, Team apps,
prevailing-wage) had become whole-box-clickable; reverted those so only the checkbox toggles.
Everything merged to main and verified byte-identical live on salescommand.app. Nothing deferred
on the checkbox work — it's fully shipped.

- Reused the already-parked branch feat/deposit-checkbox-fill instead of the new duplicate the
  /detach created — the registry already had a near-identical parked stream (deposit-checkbox-fill,

  same hash as a local build of merged main = provably the same code. `cmp` byte-check both times.
- WTCCalculator uses a LOCAL `T` palette with gray-ish names that are actually linen values
  (T.gray50 = #b5a896 = C.linen, T.green = #30cfac = teal). Left that pre-existing style-rule
  debt alone beyond the checkboxes; the new component (imports real C tokens) renders consistently.
