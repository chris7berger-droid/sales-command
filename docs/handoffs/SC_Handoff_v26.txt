====================================================
SIGNING FLOW ARCHITECTURE (SC-30)
====================================================

FLOW:
Proposal Detail → Send Proposal → ProposalPDFModal (Send tab)
→ send-proposal Edge Function → Resend email → customer gets link
→ /sign/:token → PublicSigningPage.jsx → customer types name
→ jsPDF generates branded PDF → canvas renders cursive signature
→ PDF uploaded to signed-proposals bucket
→ pdf_url saved to proposal_signatures
→ proposals.status = Sold
→ call_log.stage = Sold (synced in PublicSigningPage.jsx)
→ Proposal Detail shows ⬇ Download Signed PDF button

KEY FILES:
- PublicSigningPage.jsx — customer signing page + PDF generation
- Proposals.jsx — ProposalPDFModal (send tab) + Download button
- supabase/functions/send-proposal — email delivery Edge Function

EDGE FUNCTION SETTINGS (critical):
- send-proposal: "Verify JWT with legacy secret" = OFF
  (do not turn back on — was causing 401 errors)
