2. Password reset race condition — FIXED
   - Bug: clicking a password reset link redirected to home page instead of
     showing the "set new password" form
   - Root cause: App.jsx cleared the #type=recovery hash from the URL before
     Login.jsx had a chance to mount and read it. Login never saw the recovery
     flag, so it stayed on the default sign-in form. Then Supabase auth state
     changes bounced the user to the home page.
   - Fix: App.jsx now stashes a sessionStorage flag (sc_recovery_mode) before
     clearing the hash. Login.jsx checks both the hash AND the sessionStorage
     flag on mount, so it reliably switches to the reset form.

3. Invite link investigation
   - The invite uses a Supabase magic link (type=magiclink) which auto-signs
     the user in and shows the WelcomeScreen (password creation).
   - Denise's invite link likely expired before she clicked it (default
     Supabase magic link expiry is 1 hour).
   - NOT a code bug — but needs Supabase Dashboard config check (see below).
