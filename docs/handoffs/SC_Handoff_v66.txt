WHAT WAS DONE
===============================================================================

1. Fixed WTC print — blank page (content not printing)
   - File: src/pages/WTCCalculator.jsx
   - Root cause: the data-wtc-print-only div was NESTED inside the
     data-wtc-no-print div. The print CSS hides [data-wtc-no-print] with
     display:none !important, which also hid everything inside it —
     including the print layout.
   - Fix: moved the closing </div> for data-wtc-no-print so it closes
     before the print-only section. Now they are siblings, not parent/child.

2. Fixed material name missing on WTC print (showed "—" for all rows)
   - File: src/pages/WTCCalculator.jsx (print layout, ~line 2089)
   - Root cause: print layout referenced m.name but the materials data
     model stores the name in m.product (set at line 535 when adding
     from DB or custom).
   - Fix: changed to m.product || m.name || "—"

3. Allow decimal quantities in materials tab
   - File: src/pages/WTCCalculator.jsx (MaterialsTab, updateItem)
   - Root cause: updateItem coerced all number fields with parseFloat(val)||0,
     which collapsed "0." to 0 mid-typing, making it impossible to enter
     values like 0.5 (half kit).
   - Fix: updateItem now stores the raw string if the value ends with "."
     (mid-decimal), only coercing to number when input is complete.
     calcMaterialRow already uses parseFloat() internally so it handles
     string values safely.

4. Widened Qty and Tax % input columns in materials table
   - Both inputs widened from 55px to 65px so "8.25" is fully visible
     and decimal qty values have room to display.
