get_user_tenant_id() — HOW IT WORKS NOW
===============================================================================

Old (single-tenant assumption):
  SELECT id FROM public.tenant_config LIMIT 1;

New (auth-aware with fallback):
  SELECT COALESCE(
    (SELECT tenant_id FROM public.team_members WHERE auth_id = auth.uid() LIMIT 1),
    (SELECT id FROM public.tenant_config LIMIT 1)
  );

This means every RLS policy across every table now properly resolves the
tenant based on who is logged in. The LIMIT 1 fallback handles edge cases
(triggers, migrations, anonymous contexts).
