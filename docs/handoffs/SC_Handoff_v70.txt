WHAT WAS DONE
===============================================================================

1. DROPPED create_job_on_sold TRIGGER (DB CHANGE — LIVE)

   The trigger on the proposals table was auto-creating job rows when a
   proposal status changed to "Sold". It caused the status insert bug
   (Parked arriving as Ongoing) because the trigger hardcoded
   status: 'Ongoing'. It also created incomplete job rows missing
   field_sow, materials, size, source_proposal_id, and other fields
   that the manual Send to Schedule flow provides.

   - Dropped trigger: trg_create_job_on_sold
   - Dropped function: public.create_job_on_sold()
   - Manual "Send to Schedule" in ProposalDetail.jsx is now the SOLE
     path for creating jobs in the jobs table
   - jobs.status column default is still 'Ongoing' (harmless — manual
     insert always explicitly sets 'Parked')

2. MATERIALS TRANSFER ON SEND TO SCHEDULE

   handleSendToSchedule() now creates materials rows from WTC data
   after inserting the job. Replaces the material-creation logic that
   was in the now-dropped trigger.

   Format:
     name:   "Product Name (kit_size)" e.g. "Hi-Tech PE85 (10 gallon)"
     notes:  "Qty: 3 | Supplier: Hi-Tec"
     status: "Not Ordered"
     ordinal: sequential across all WTCs

   Materials insert failure is logged but non-blocking.

3. PROPOSAL NUMBER + CHANGE ORDER FIELDS ON JOBS TABLE

   Added columns to public.jobs (DB CHANGE — LIVE):
     - proposal_number (integer) — P1, P2, P3 sequence from proposals
     - is_change_order (boolean, default false)
     - co_number (integer)

   handleSendToSchedule() now passes:
     - p.proposal_number
     - p.call_log.is_change_order
     - p.call_log.co_number

   Also added is_change_order and co_number to the call_log select
   join in ProposalDetail.jsx (3 occurrences).

7. EMAIL FROM SALES REP (send-proposal edge function — DEPLOYED LIVE)

   Proposal emails now send FROM the sales rep's actual email address
   when their domain is verified in Resend. Previously hardcoded
   noreply@salescommand.app for all proposals.

   Verified domains: hdspnv.com, scmybiz.com, schmybiz.com, salescommand.app

   Logic:
     const repDomain = repEmail.split("@")[1]?.toLowerCase()
     if VERIFIED_DOMAINS.includes(repDomain) → from: "Rep Name <rep@domain>"
     else → from: "Rep Name <noreply@salescommand.app>"

   send-proposal was broken (always noreply).
