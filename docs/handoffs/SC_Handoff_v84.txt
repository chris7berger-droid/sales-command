OPEN BUG DOCUMENTED FOR LATER
===============================================================================

INVOICE SEND ERROR SURFACING (caught during testing, deferred)
  - When supabase.functions.invoke("send-invoice", ...) returns 4xx/5xx,
    the JSON body containing the real error ("Invalid invoice amount",
    "Customer email is required", etc.) is swallowed by supabase-js.
  - User sees a generic "Send failed." (or sometimes nothing) and has to
    open DevTools to find the 400.
  - Fix: in InvoicePDFModal.handleSend (Invoices.jsx ~line 565), wrap the
    invoke and parse the response body manually on non-2xx, OR switch to
    a direct fetch() so we can read the body regardless of status.
  - Same pattern likely affects qb-sync-invoice and other edge fn calls
    — sweep when fixing.
