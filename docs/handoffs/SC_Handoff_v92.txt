OPEN BUG — QuickBooks "Connection Failed" (403)
────────────────────────────────────────────────
Reported at end of session. QB OAuth callback page (/qb/callback) gets a 403
when calling qb-auth edge function with action "exchange".

Console evidence:
  - 403 on qb-auth edge function
  - gotrue-js lock timeout: auth token refresh stuck for 5s, forcefully recovered

Investigation findings:
  - Settings handleSave does NOT touch QB data — qb_connection is a separate
    table, never written by Settings
  - qb-auth uses requireAdminOrManager() for exchange — 403 comes from
    tenantAuth.ts (no team membership, inactive, no tenant, or wrong role)
  - The gotrue-js lock timeout means auth session was stale when user returned
    from QB's OAuth redirect
  - qb-auth was last deployed May 2 (security audit) — unclear if deployed
    with --no-verify-jwt
  - NOT caused by any changes in this session

Most likely fix:
  1. Redeploy qb-auth with --no-verify-jwt (gateway rejecting stale JWT)
  2. May also need retry/await-session logic in QBCallbackPage.jsx

Key files:
  - supabase/functions/qb-auth/index.ts
  - supabase/functions/_shared/tenantAuth.ts
  - src/pages/QBCallbackPage.jsx
  - src/pages/Settings.jsx (QBIntegrationCard at line 382)
