   - NewPayAppModal wrote retainage_pct/retainage_amount, which don't exist
     on the invoices table — actual columns are retention_pct/retention_amount.
     Values silently dropped, retention card never appeared.
   - Fixed insert + handleSaveLines (also updates retention_amount on % edit)
     + read of retainage_amount → retention_amount in detail modal.
   - Files: src/components/NewPayAppModal.jsx, src/components/PayAppDetailModal.jsx

5. QB Sync Routed Retention to Discount Account
   - qb-sync-invoice was using DiscountLineDetail for retention — math was
     right (net = gross - retention), but $ landed in QB's default discount
     account, not the construction-standard Other Current Asset bucket.
   - Replaced with negative SalesItemLineDetail using the tenant's existing
     "1121- Retention %" service item (mapped to Construction in
     Progress-Retent in HDSP's books).
   - Lookup tries "1121- Retention %" → "Retention" → "Retainage". Falls
     back to DiscountLineDetail with warning if none exist.
   - File: supabase/functions/qb-sync-invoice/index.ts (deployed)

6. No Sync to QB Path on Pay App Invoice Detail

QB RETENTION STRATEGY (decision + implementation)
─────────────────────────────────────────────────
Sales Command pushes pay-app invoices as **gross + negative retainage line**.

For HDSP today:
- Retention item: "1121- Retention %" (Service) → Construction in
  Progress-Retent (Other Current Asset).
- Result in QB: A/R balance = net Payment Due ($40,936 on AGRU Inv #10024,
  not $43,091). Held retention ($2,154.53) accrues to Construction in
  Progress-Retent.
- A/R Aging won't show retention as delinquent.
