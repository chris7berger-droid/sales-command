SECURITY FIXES
──────────────
1. H1/H7 — SET search_path on security functions
   - Functions: public.get_user_tenant_id, public.request_signing_token,
     public.request_viewing_token
   - get_user_tenant_id is SECURITY DEFINER → real CVE-2018-1058 vector;
     prod was already partially hardened (proconfig had search_path=public)
     per pre-apply pg_proc query, so the ALTER on that one was a no-op.
   - request_signing_token / request_viewing_token are STABLE (not
     SECURITY DEFINER) → defense-in-depth; both went from proconfig=null
     to {search_path=public}.
   - Pre-apply pg_proc output captured in the migration header for
     future provenance.
   - Verified post-apply: all 3 proconfig=["search_path=public"] on prod.

2. H6 — public signing page no longer exposes pricing internals
   - PublicSigningPage previously did supabase.from('proposal_wtc')
     .select('*') so it could call calcWtcPrice(wtc) client-side. The
     wildcard returned the full cost stack to the customer's browser.
   - Architecture: store per-WTC total at lock time, read it back via
     SECURITY DEFINER RPC. Avoids duplicating calcWtcPrice business
     logic in SQL (which would create a silent-drift surface — customer
     sees price X on public page while internal app shows price Y).
   - New column proposal_wtc.locked_line_total (nullable numeric).
   - handleLock writes it in three places:
       toggleWtcLock — src/components/ProposalDetail.jsx:233-247
       handlePullBack — src/components/ProposalDetail.jsx:467-470
         (clears to NULL when pulled back)
       handleLock — src/pages/WTCCalculator.jsx:1781-1810
   - New RPC public.get_public_proposal_view(p_token) — SECURITY DEFINER,
     SET search_path = public, GRANT EXECUTE TO anon. Returns json with
     proposal core + call_log + customers + per-WTC {id, sales_sow,
     locked_line_total, work_type_name}. No cost basis ever returned.
     Pattern matches v91 mark_proposal_signed / mark_recipient_viewed.
   - PublicSigningPage rewired: one rpc('get_public_proposal_view') call
     replaces both the proposals select('*, …') and the proposal_wtc
     select('*, work_types(name)'). Grand total reads proposal.total
     (authoritative). Per-WTC totals read locked_line_total. calcWtcPrice
     import removed.
   - Backfill: scripts/backfill_locked_line_total.mjs imports the
     canonical calcWtcPrice and writes locked_line_total for already-
     locked WTCs. Run once after migration deploy with --apply.
     Result on prod: 127 rows written, 0 failed.
   - Vercel preview smoke test passed: TEST proposal locked, sent,
     signed via incognito; per-WTC totals + grand total matched internal
     app; network-tab payload for rpc/get_public_proposal_view contained
     no burden_rate / markup_pct / materials.
