   - Lesson saved as feedback memory
     `feedback_parallel_session_collisions.md` and indexed in MEMORY.md:
       * Make edit → git add → git commit in one tight sequence; do
         not pause with a dirty tree when another session is active.
       * If a branch switch is unavoidable mid-edit, run
         `git stash push -m "<descriptive>" -- <paths>` myself first;
         do not trust git's carry-uncommitted-changes behavior to
         survive contact with a parallel session.
       * After any unexpected branch state shift, check `git stash
         list` and `git fsck --lost-found` before assuming work is gone.
